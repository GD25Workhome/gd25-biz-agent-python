# 先思考
## 你的纲领
- 改写一致性：改写后的提问/回复最好在风格、术语、长度上有明确规范（如写进 prompt 或示例），避免同一场景下表述差异过大，影响检索与人工评估。
- 标签体系：先定一个小而稳的标签集（含场景、意图、质量档位、领域、子领域、情绪、复杂度等），再逐步扩展；避免一开始标签过多导致标注不一致。
- 质量初筛：在模型输出后加一层规则或轻量模型过滤（如改写失败、过短、敏感词），再进入人工，可节省人工时间。
- 可解释性：若模型能输出「改写依据」或「场景置信度」，有利于人工复核与纠错。
- 其它：
  - 文本处理：编码、换行、特殊字符、html标签等处理
  - 去重：基于语义或文本指纹，如 MinHash + LSH
  - 工具：Apache NiFi, Airflow, Pandas, Spark

# 历史会话 Q/A 解析与空字符兼容改造设计

> 针对 `sh-1128_副本` 的「患者无数据+历史会话+历史Action」Sheet，解决空字符判断与「历史会话」字段 Q/A 格式解析问题。

---

## 一、问题背景

### 1.1 目标 Sheet

- **Excel**：`sh-1128_副本.xlsx`
- **Sheet**：`患者无数据+历史会话+历史Action`
- **当前解析策略**：`sh1128`（通过 `excel_parser_mapping`）

### 1.2 问题描述

#### 问题 1：空字符判断需增加 "-" 兼容

运行中发现，Excel 中部分单元格使用 `-` 表示空值，当前逻辑未将其视为空，导致：
- 本应跳过的空行未被跳过
- 本应视为 None 的字段被当作有效值处理

**期望**：`-`、`--`、`---` 等仅由连字符组成的值，应视为空。

#### 问题 2：「历史会话」字段为 Q/A 格式，需拆分为多历史消息

「历史会话」字段实际值示例：

```
Q：小悦您好！我想约龙医生，做造影
A：您有和医生提前预约吗？
Q：没有
A：了解了，方便告知下患者就诊姓名、诊疗卡号、住院时间、具体什么情况吗？
----
```

**当前逻辑**：`build_history_messages(history_session, history_response)` 将 `历史会话`、`历史会话响应` 各作为**单条** human/ai 消息。

**期望**：将上述 Q/A 文本解析为多条交替的 human/ai 消息：

```python
[
    {"type": "human", "content": "小悦您好！我想约龙医生，做造影"},
    {"type": "ai", "content": "您有和医生提前预约吗？"},
    {"type": "human", "content": "没有"},
    {"type": "ai", "content": "了解了，方便告知下患者就诊姓名、诊疗卡号、住院时间、具体什么情况吗？"}
]
```

## 二、现状分析

### 2.1 空字符判断现状

| 函数 | 当前空值判断 | 涉及场景 |
|------|--------------|----------|
| `convert_to_text` | `pd.isna`、`None`、`""`、`"无"` | 文本字段 |
| `convert_to_string` | 同上 | 字符串字段 |
| `convert_to_int` | 同上 | 数值字段 |
| `extract_message_id` / `extract_message_ids` | 同上 | message_id |
| `is_empty_row` | 依赖 `convert_to_text` | 空行判断 |

**缺失**：`-`、`--`、`---` 等未被视为空。

### 2.2 历史会话处理现状

| 环节 | 当前逻辑 | 数据流 |
|------|----------|--------|
| `extract_row_data` | 提取 `历史会话`、`历史会话响应` 为原始字符串 | row_data |
| `build_history_messages` | 单条 human + 单条 ai | `[human, ai]` |
| `to_dataset_item` | 使用 `history_session` / `history_response` 或 `history_messages` | input.history_messages |

**缺失**：无法将「历史会话」中的 Q/A 交替格式解析为多轮 `history_messages`。

### 2.3 数据流示意

```
Excel 行
  → extract_row_data: history_session = "Q：...\nA：...\nQ：...\nA：..."
  → to_dataset_item: build_history_messages(history_session, history_response)
      当前：将整段 history_session 作为一条 human 消息 ❌
      期望：解析为 [human, ai, human, ai, ...] ✅
```

## 三、改造方案

### 3.1 空字符 "-" 兼容

#### 3.1.1 方案：新增「视为空」的字符串集合

在 `parsers/base.py` 中：

1. **定义空值常量**（可配置）：

```python
# 视为空的字符串（strip 后）
_EMPTY_VALUES = frozenset({"", "无", "-", "—", "–"})
# 可选：仅由连字符、空格组成的也视为空
def _is_empty_like(s: str) -> bool:
    if not s:
        return True
    t = s.strip()
    if not t:
        return True
    if t in _EMPTY_VALUES:
        return True
    # 仅由 - 组成
    if t.replace("-", "").replace("—", "").replace("–", "").strip() == "":
        return True
    return False
```

2. **修改 `convert_to_text`、`convert_to_string`**：在判断空值时调用 `_is_empty_like`。

3. **修改 `convert_to_int`**：同上。

4. **`is_empty_row`**：依赖 `convert_to_text`，无需单独改，随 `convert_to_text` 生效。

#### 3.1.2 影响范围

| 函数 | 修改方式 |
|------|----------|
| `convert_to_text` | 增加 `_is_empty_like(result)` 判断 |
| `convert_to_string` | 同上 |
| `convert_to_int` | 同上 |
| `extract_message_ids` | 增加 `_is_empty_like(value_str)` 判断 |

### 3.2 历史会话 Q/A 解析

#### 3.2.1 方案：新增 `parse_history_qa_to_messages`

在 `parsers/base.py` 中新增：

```python
def parse_history_qa_to_messages(text: Optional[str]) -> List[Dict[str, str]]:
    """
    将「历史会话」Q/A 格式解析为 BaseMessage 数组。

    格式：Q：xxx / Q: xxx、A：xxx / A: xxx 交替出现。
    支持末尾分隔符如 ----。

    Returns:
        List[Dict]: [{"type":"human","content":"Q1"}, {"type":"ai","content":"A1"}, ...]
    """
```

**解析规则**：

| 规则 | 说明 |
|------|------|
| 分隔符 | `Q：`、`Q:`、`A：`、`A:`（中英文冒号） |
| 顺序 | 按出现顺序交替，Q→A→Q→A |
| 末尾分隔符 | `----`、`---` 等可忽略 |
| 单块 | 若无法解析为 Q/A，整段作为 human 消息（兼容旧格式） |

**实现思路**：复用 `parse_qa_blocks` 的匹配逻辑，同时提取 A 块，按顺序组装 `[human, ai, human, ai, ...]`。

#### 3.2.2 与 `build_history_messages` 的集成

**策略 A：在 `build_history_messages` 内分支**

```python
def build_history_messages(
    history_session: Optional[str],
    history_response: Optional[str],
    *,
    parse_qa_format: bool = False,
) -> List[Dict[str, Any]]:
    if parse_qa_format and history_session:
        parsed = parse_history_qa_to_messages(history_session)
        if parsed:
            return parsed
    # 原有逻辑
    messages = []
    if history_session:
        messages.append({"type": "human", "content": history_session})
    if history_response:
        messages.append({"type": "ai", "content": history_response})
    return messages
```

**策略 B：在解析策略层决定是否解析 Q/A**

在 `extract_row_data` 或 `to_dataset_item` 中，由解析策略决定：
- 若 `历史会话` 包含 `Q：`/`Q:` 且策略支持，则调用 `parse_history_qa_to_messages`
- 否则走原有 `build_history_messages`

**推荐**：策略 B，通过解析策略控制，避免影响其他 Sheet。

#### 3.2.3 解析策略扩展

为「患者无数据+历史会话+历史Action」新增或复用解析策略：

| 方案 | 说明 |
|------|------|
| 方案 1 | 新增 `Sh1128HistoryQAParser`，继承 Sh1128Parser，重写 `extract_row_data` 或 `to_dataset_item` 中 history 处理 |
| 方案 2 | 在 Sh1128Parser 中增加「历史会话 Q/A 解析」开关，由 Config 或 sheet 名控制 |
| 方案 3 | 新增 `parse_history_qa_to_messages`，在 `to_dataset_item` 中：若 `history_session` 包含 `Q：`/`Q:` 则自动解析 |

**推荐**：方案 1，清晰、可扩展，不影响现有 sh1128 行为。

### 3.3 实现路径

| 阶段 | 内容 | 涉及文件 |
|------|------|----------|
| 1 | 新增 `_is_empty_like`，修改 `convert_to_text`、`convert_to_string`、`convert_to_int`、`extract_message_ids` | `parsers/base.py` |

| 5 | 注册新解析器 | `parsers/strategies/__init__.py` |

## 四、设计细节

### 4.1 `parse_history_qa_to_messages` 解析逻辑

**输入示例**：

```
Q：小悦您好！我想约龙医生，做造影
A：您有和医生提前预约吗？
Q：没有
A：了解了，方便告知下患者就诊姓名、诊疗卡号、住院时间、具体什么情况吗？
----
```

**步骤**：

1. 去除末尾 `----`、`---` 等分隔符
2. 使用正则按 `Q：`/`Q:`、`A：`/`A:` 切分，保留顺序
3. 交替组装：第 1 块为 human，第 2 块为 ai，第 3 块为 human，…
4. 若无法切分（无 Q/A 标记），整段作为一条 human 消息

**边界**：

- 空或 None → `[]`
- 仅 Q 无 A → 最后一条为 human
- 仅 A 无 Q → 可视为 ai 开头，或整段作 human（按业务约定）

### 4.2 空字符 "-" 判断规则

| 值 | 是否视为空 |
|----|------------|
| `None`、`pd.NA` | 是 |
| `""`、`"   "` | 是 |
| `"无"` | 是 |
| `"-"` | 是 |
| `"--"`、`"---"`、`"----"` | 是 |
| `" - "` | 是（strip 后为 `-`） |
| `"—"`、`"–"`（全角、en-dash） | 是（可选） |
| `"- 有内容"` | 否 |

## 五、兼容性

- **空字符**：所有使用 `convert_to_text`、`convert_to_string`、`convert_to_int` 的解析策略均受益，无破坏性变更。
- **历史会话 Q/A**：仅对使用 `sh1128_history_qa` 的 Sheet 生效，其他 Sheet 保持原逻辑。

## 六、测试要点

1. **空字符**：`convert_to_text("-")`、`convert_to_text("---")` 返回 `None`
2. **历史会话解析**：`parse_history_qa_to_messages` 对示例输入返回 4 条消息，交替 human/ai
3. **兼容**：`convert_to_text("正常内容")` 行为不变
4. **Sh1128HistoryQAParser**：集成测试，一行数据经解析后 `history_messages` 为多轮格式

# 医生排班信息嵌入方案设计

## 一、需求分析

### 1.1 目标
在 `50-core_agent.md` 的 `{llm_context_part}` 占位符处追加医生排班信息，让模型在回答问题时可以使用该信息。

### 1.2 当前数据流

#### 1.2.1 医生信息构建（login.py）
- **位置**：`backend/app/api/routes/login.py:230-234`
- **构建逻辑**：
  ```python
  doctor_info = {
      "doctor_id": doctor_id,
      "doctor_name": "张医生",
      "schedule": generate_doctor_schedule(days=14)  # 生成14天的排班信息
  }
  ```
- **存储位置**：`session_context["doctor_info"]`
- **数据结构**：`schedule` 为 `List[Dict[str, Any]]`，每个元素包含：
  - `date`: 日期字符串（YYYY-MM-DD）
  - `morning`: 上午时间段（8:00-12:00），可选
  - `afternoon`: 下午时间段（14:00-18:00），可选

#### 1.2.2 上下文信息模板（llm_context_part.md）
- **位置**：`config/flows/flow_rule/llm_context_part.md`
- **当前内容**：
  ```markdown
  - 当前日期： 
      {current_date}
  - 患者基础信息： 
      {user_info}
  - 其它信息（如有会填写，否则忽略）：
      {other_info}
  ```
- **替换机制**：
  1. `{llm_context_part}` 占位符在 `prompt_manager._replace_flow_rule_placeholders` 中被替换为 `llm_context_part.md` 的内容
  2. `{current_date}`、`{user_info}`、`{other_info}` 在 `build_system_message` 中从 `prompt_vars` 替换

#### 1.2.3 占位符替换流程
1. **flow_rule 占位符替换**（prompt_manager.py）：
   - 在 `cached_prompt` 时，调用 `_replace_flow_rule_placeholders`
   - 将 `{llm_context_part}` 替换为 `llm_context_part.md` 的内容
   - 此时 `{current_date}`、`{user_info}`、`{other_info}` 仍为占位符

2. **prompt_vars 占位符替换**（sys_prompt_builder.py）：
   - 在 `build_system_message` 时，从 `state.prompt_vars` 和 `state.edges_var["edges_prompt_vars"]` 读取值
   - 替换 `{current_date}`、`{user_info}`、`{other_info}` 等占位符

#### 1.2.4 当前数据来源
- **current_date**：在 `build_initial_state` 中设置到 `prompt_vars["current_date"]`
- **user_info**：在 `build_initial_state` 中从 `token_context` 读取，设置到 `prompt_vars["user_info"]`
- **other_info**：在 `query_user_info_node` 中查询血压数据，设置到 `prompt_vars["other_info"]`

## 二、方案设计

### 方案一：在 build_initial_state 中设置 doctor_schedule（推荐）

#### 2.1.1 方案描述
在 `build_initial_state` 中从 `session_context` 读取 `doctor_info.schedule`，格式化后设置到 `prompt_vars["doctor_schedule"]`，然后在 `llm_context_part.md` 中添加 `{doctor_schedule}` 占位符。

#### 2.1.2 实现步骤

**步骤1：修改 helpers.py - build_initial_state**
- 在 `build_initial_state` 中获取 `session_context`
- 从 `session_context["doctor_info"]["schedule"]` 读取排班信息
- 格式化排班信息为可读文本（如 Markdown 列表）
- 设置到 `prompt_vars["doctor_schedule"]`

**步骤2：修改 llm_context_part.md**
- 在 `{other_info}` 后添加医生排班信息部分：
  ```markdown
  - 医生排班信息（如有会填写，否则忽略）：
      {doctor_schedule}
  ```

**步骤3：格式化函数**
- 创建 `_format_doctor_schedule(schedule: List[Dict]) -> str` 函数
- 将排班列表格式化为易读的文本格式

#### 2.1.3 优点
- ✅ **职责清晰**：`build_initial_state` 负责构建初始状态，包括从 session_context 读取数据
- ✅ **统一管理**：所有 prompt_vars 都在 `build_initial_state` 中设置，便于维护
- ✅ **无需修改节点**：不需要修改 `query_user_info_node` 或其他节点
- ✅ **数据流清晰**：session_context → prompt_vars → build_system_message → 占位符替换
- ✅ **易于测试**：可以在 `build_initial_state` 中直接测试格式化逻辑

#### 2.1.4 缺点
- ⚠️ **格式化逻辑位置**：需要在 helpers.py 中添加格式化函数（或抽取到独立模块）

#### 2.1.5 代码改动点
1. **backend/app/api/helpers.py**：
   - 在 `build_initial_state` 中添加读取 `session_context["doctor_info"]` 的逻辑
   - 添加 `_format_doctor_schedule` 函数
   - 将格式化后的排班信息设置到 `prompt_vars["doctor_schedule"]`

2. **config/flows/flow_rule/llm_context_part.md**：
   - 添加医生排班信息部分和 `{doctor_schedule}` 占位符

## 四、推荐方案

**推荐方案一**：在 `build_initial_state` 中设置 `doctor_schedule`

### 4.1 推荐理由
1. **职责清晰**：`build_initial_state` 负责构建初始状态，从 `session_context` 读取数据符合其职责
2. **统一管理**：所有 `prompt_vars` 都在同一处设置，便于维护和调试
3. **数据流清晰**：session_context → prompt_vars → build_system_message → 占位符替换
4. **无需修改节点**：不破坏现有节点逻辑
5. **易于测试**：格式化逻辑可以独立测试

### 4.2 实现细节

#### 4.2.1 格式化函数设计
```python
def _format_doctor_schedule(schedule: List[Dict[str, Any]]) -> str:
    """
    格式化医生排班信息为可读文本
    
    Args:
        schedule: 排班信息列表
        
    Returns:
        str: 格式化后的文本（Markdown 格式）
    """
    if not schedule:
        return ""
    
    lines = []
    for item in schedule:
        date = item.get("date", "")
        morning = item.get("morning")
        afternoon = item.get("afternoon")
        
        day_info = f"- {date}: "
        periods = []
        if morning:
            periods.append("上午 8:00-12:00")
        if afternoon:
            periods.append("下午 14:00-18:00")
        
        if periods:
            day_info += "、".join(periods)
        else:
            day_info += "休息"
        
        lines.append(day_info)
    
    return "\n".join(lines)
```

#### 4.2.2 build_initial_state 修改点
```python
# 在 build_initial_state 中，设置 prompt_vars 后添加：

# 获取 session_context
session_context = context_manager.get_session_context(request.session_id)
if session_context:
    doctor_info = session_context.get("doctor_info")
    if doctor_info and "schedule" in doctor_info:
        schedule = doctor_info["schedule"]
        formatted_schedule = _format_doctor_schedule(schedule)
        prompt_vars["doctor_schedule"] = formatted_schedule
    else:
        prompt_vars["doctor_schedule"] = ""
else:
    prompt_vars["doctor_schedule"] = ""
```

#### 4.2.3 llm_context_part.md 修改
```markdown

    {current_date}
- 患者基础信息： 
    {user_info}
- 其它信息（如有会填写，否则忽略）：
    {other_info}
- 医生排班信息（如有会填写，否则忽略）：
    {doctor_schedule}
```

## 五、实施计划

### 5.1 实施步骤
1. **步骤1**：在 `helpers.py` 中添加 `_format_doctor_schedule` 函数
2. **步骤2**：修改 `build_initial_state`，添加读取和设置 `doctor_schedule` 的逻辑
3. **步骤3**：修改 `llm_context_part.md`，添加医生排班信息部分
4. **步骤4**：测试验证：
   - 验证 `doctor_schedule` 正确设置到 `prompt_vars`
   - 验证 `{doctor_schedule}` 占位符正确替换
   - 验证格式化后的文本可读性

### 5.2 注意事项
1. **空值处理**：如果 `doctor_info` 或 `schedule` 不存在，应设置为空字符串，避免占位符未替换
2. **格式化一致性**：确保格式化后的文本与 `other_info` 的格式风格一致
3. **性能考虑**：格式化逻辑应高效，避免在每次请求时重复计算（可以考虑缓存）

## 六、备选方案说明

如果未来需要更灵活的医生信息展示（如支持多种格式、动态配置等），可以考虑：
- **方案四**：将医生信息独立成 flow_rule 文件，便于单独维护和扩展
- **混合方案**：结合方案一和方案四，在 `build_initial_state` 中设置，但在独立的 flow_rule 文件中定义模板

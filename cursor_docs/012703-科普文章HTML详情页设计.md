# 科普文章 HTML 详情页设计

**文档编号**：012703  
**日期**：2026-01-27  
**范围**：为 `PopularScienceArticle` 提供基于主键 ID 的独立 HTML 展示页面，支撑「回复中引用文章链接」能力。

---

## 一、需求与目标

- **需求背景**：
  - 已有科普文章表 `PopularScienceArticle`，字段 `article_content` 存储「文章详情 HTML 内容」。
  - 上游 RAG 流程希望在回复中输出「可点击的文章链接」，点击后直接在浏览器中打开文章详情页面。
- **目标**：
  - 提供一个 **基于文章主键 ID 的独立 HTML 页面**。
  - 当前实现路径：`/api/v1/articles/{id}`，直接返回完整 HTML，而非 JSON。
  - 页面样式简洁、可读性好，适合 H5 / WebView 打开。

---

## 二、后端接口设计

- **框架**：FastAPI
- **路由文件**：`backend/app/api/routes/articles.py`
- **聚合入口**：`backend/app/api/routes/__init__.py`

### 2.1 路由信息

- **HTTP 方法**：`GET`
- **URL 模式**：`/api/v1/articles/{id}`
- **路径参数**：
  - `id: int` —— `PopularScienceArticle` 的主键 ID（`id` 字段）。
- **返回类型**：
  - `fastapi.responses.HTMLResponse`，直接返回完整 HTML 文档。

### 2.2 依赖与模型

- **数据库模型**：`backend/infrastructure/database/models/rag_models.py` 中的 `PopularScienceArticle`
  - 关键字段：
    - `id: Integer` —— 主键 ID。
    - `article_title: Text` —— 文章标题。
    - `article_content: Text` —— 文章详情内容（约定为 HTML 片段）。
- **数据库会话**：
  - 使用已存在的依赖：`get_async_session`（`backend/infrastructure/database/connection.py`）。
  - 类型：`AsyncSession`（SQLAlchemy 异步会话）。

### 2.3 接口核心逻辑

1. **根据 ID 查询文章**
   - 使用 `session.get(PopularScienceArticle, article_id)` 按主键查询。
   - 若记录不存在：
     - 抛出 `HTTPException(status_code=404, detail="科普文章不存在，id=...")`。

2. **数据安全与渲染策略**
   - 标题：
     - 使用 `html.escape(article.article_title)` 做 HTML 转义，防止标题被注入为 HTML 或脚本。
   - 正文内容：
     - 直接使用 `article.article_content` 作为 HTML 片段嵌入页面。
     - 约定此字段来源可信（导入脚本保证），当前不额外做 HTML 清洗。
     - 若未来需要更严格的安全策略，可在此位置插入 HTML Sanitizer。

3. **完整 HTML 页面结构**
   - 基本结构：
     - `<!DOCTYPE html>` + `<html lang="zh-CN">`。
     - `<head>` 中包含：
       - `meta charset="UTF-8"`。
       - `meta viewport`（自适应移动端）。
       - `<title>` 使用转义后的文章标题。
       - 内联 `style`，提供简洁现代的阅读样式。
     - `<body>` 中：
       - 外层容器 `.page-container` 控制最大宽度和居中。
       - 卡片 `.article-card` 提供白底 + 阴影效果。
       - `.article-title` 显示标题。
       - `.article-content` 容器内直接插入 `article_content` HTML 内容。
   - 样式要点：
     - 使用系统常见无衬线字体族，提升跨平台一致性。
     - 控制行高与段落间距，保证可读性。
     - 为 `a` 标签定义主色（如 `#2563eb`），并在 `:hover` 时显示下划线。

4. **异常处理与日志**
   - 已知业务异常：
     - `HTTPException` 直接抛出交由 FastAPI 处理。
   - 未预期异常：
     - `await session.rollback()` 回滚当前事务。
     - 使用 `logger.error(..., exc_info=True)` 记录详细堆栈。
     - 返回 `HTTPException(status_code=500, detail="渲染科普文章页面失败，请稍后重试。")`。

## 三、与 RAG 流程 / 前端的对接约定

### 3.1 链接拼接建议

- 当前接口路径：`/api/v1/articles/{id}`。
- 上游 RAG / 提示词中，可以按如下形式构造 Markdown 链接：
  - 示例：`[文章标题](https://{域名或网关}/api/v1/articles/{id})`。
- 后续若需要与 `article_material_id` 绑定专用路由（例如 `/article/{article_material_id}`）：
  - 可以在本接口之外另增一层「ID 映射路由」，本次实现不做强绑定，仅以主键 ID 为准。

### 3.2 使用场景

- **在回复中引用文章**：
  - RAG 流程返回结构中包含文章 ID 和标题。
  - 模板层将其渲染为 Markdown 链接列表：
    - 例如：
      - `- [高血压用药基础知识](https://.../api/v1/articles/123)`
      - `- [家庭血压监测要点](https://.../api/v1/articles/456)`
  - 前端渲染 Markdown 后，点击链接即跳转到本 HTML 详情页。

## 四、扩展与后续优化方向

1. **基于 `article_material_id` 的路由**
   - 可新增：
     - `/api/v1/articles/by-material/{article_material_id}`
   - 内部根据 `article_material_id` 查询对应记录，再复用同一套 HTML 渲染逻辑。

2. **安全与内容清洗**
   - 若未来文章 HTML 来源不再完全可信，可：
     - 在写入 `article_content` 前做统一清洗。
     - 或在本接口中引入 HTML Sanitizer，白名单化标签与属性。

3. **埋点与追踪**
   - 可在页面中：
     - 预留 `<script>` 区域，挂载埋点 SDK。
     - 通过查询参数（如 `?from=medical_agent_v6_1&session_id=...`）记录来源。

4. **样式与主题扩展**
   - 目前样式为简洁通用版：
     - 后续可：
       - 抽取为单独 CSS 文件，前后端统一维护。
       - 增加暗色模式 / 字号调节等易用性功能。

## 五、测试要点（后端接口）

1. **存在的文章 ID**
   - 准备一条 `PopularScienceArticle` 记录，记录其 `id`。
   - 访问 `/api/v1/articles/{id}`：
     - 返回状态码 `200`。
     - 页面 `<title>` 与页面内标题与数据库中 `article_title` 一致。
     - 页面中包含 `article_content` 中的 HTML 结构。

2. **不存在的文章 ID**
   - 访问一个不存在的 ID：
     - 返回状态码 `404`。
     - `detail` 文案包含「科普文章不存在」字样。

3. **异常场景**
   - 模拟数据库连接异常（如关闭数据库）：
     - 接口返回 `500`。
     - 日志中能看到错误堆栈信息。

## 1. 概述

### 1.1 目标

构建数据导入管理模块，实现：
1. **数据库表设计**：基于融合版本数据结构，建立 dataSets、dataSetsItems、dataSetsPath、importConfig 四张表
2. **CRUD 管理代码**：提供完整的增删改查接口与仓储层
3. **前端管理界面**：独立「数据清洗」入口，左侧菜单 + 中间 Tab 页风格，支持数据导入管理与数据查看

**暂不实现**：数据导入的具体业务逻辑（如从 Excel、知识库等数据源导入）

### 1.2 技术栈

| 层级 | 技术选型 |
|------|----------|
| 后端 | FastAPI + SQLAlchemy（异步） + PostgreSQL |
| 前端 | Vue 3 + Element Plus + Axios |
| 数据库迁移 | Alembic |

## 2. 数据库设计

### 2.1 表结构总览

```
dataSetsPath (文件夹)  ──1:N──> dataSets (数据集合)
                                    │
                                    └──1:N──> dataSetsItems (数据项)
importConfig (导入配置) ──N:1──> 与导入逻辑强绑定（后台配置）
```

### 2.2 dataSetsPath（数据集合文件夹）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | VARCHAR(50) | PK | 可主动设置，用于路径拼接 |
| id_path | VARCHAR(500) | 可空 | 上级路径。根节点为空；下级为上级 id；多级用英文逗号拼接 |
| name | VARCHAR(200) | NOT NULL | 名称 |
| description | TEXT | 可空 | 描述 |
| metadata | JSONB | 可空 | 扩展元数据 |
| created_at | TIMESTAMPTZ | NOT NULL | 创建时间 |
| updated_at | TIMESTAMPTZ | 可空 | 更新时间 |

**索引**：`id_path`（便于按路径查询子节点）

### 2.3 dataSets（数据集合）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | VARCHAR(50) | PK | ULID 或业务 ID |
| name | VARCHAR(200) | NOT NULL | 名称 |
| path_id | VARCHAR(50) | FK | 关联 dataSetsPath.id |
| input_schema | JSONB | 可空 | input 的 JSON Schema |
| output_schema | JSONB | 可空 | output 的 JSON Schema |
| metadata | JSONB | 可空 | 扩展元数据 |
| created_at | TIMESTAMPTZ | NOT NULL | 创建时间 |
| updated_at | TIMESTAMPTZ | 可空 | 更新时间 |

**索引**：`path_id`（按文件夹查询）

### 2.4 dataSetsItems（数据项）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | VARCHAR(50) | PK | ULID |
| dataset_id | VARCHAR(50) | FK, NOT NULL | 关联 dataSets.id |
| unique_key | VARCHAR(200) | 可空 | 业务唯一 key |
| input | JSONB | 可空 | 输入数据 |
| output | JSONB | 可空 | 输出数据 |
| metadata | JSONB | 可空 | 扩展元数据 |
| status | SMALLINT | NOT NULL, DEFAULT 1 | 1=激活，0=废弃 |
| source | VARCHAR(200) | 可空 | 来源 |
| created_at | TIMESTAMPTZ | NOT NULL | 创建时间 |
| updated_at | TIMESTAMPTZ | 可空 | 更新时间 |

**索引**：`dataset_id`、`unique_key`（可选唯一索引）、`status`

### 2.5 importConfig（导入配置）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | VARCHAR(50) | PK | ULID |
| name | VARCHAR(200) | NOT NULL | 配置名称 |
| description | TEXT | 可空 | 描述 |
| import_config | JSONB | 可空 | 导入逻辑配置，由后台强绑定 |
| created_at | TIMESTAMPTZ | NOT NULL | 创建时间 |
| updated_at | TIMESTAMPTZ | 可空 | 更新时间 |

**说明**：`import_config` 为 JSON，根据实际导入类型（如 feishu_excel、knowledge_base）选择对应导入逻辑，由后台代码强绑定解析。

### 2.6 表名前缀

本模块使用前缀：`pipeline_`，即：
- `pipeline_data_sets_path`
- `pipeline_data_sets`
- `pipeline_data_sets_items`
- `pipeline_import_config`

## 3. 后端设计

### 3.1 目录结构

```
backend/
├── infrastructure/database/
│   ├── models/
│   │   ├── data_sets_path.py      # 新建
│   │   ├── data_sets.py           # 新建
│   │   ├── data_sets_items.py     # 新建
│   │   └── import_config.py      # 新建
│   └── repository/
│       ├── data_sets_path_repository.py   # 新建
│       ├── data_sets_repository.py        # 新建
│       ├── data_sets_items_repository.py  # 新建
│       └── import_config_repository.py   # 新建
├── app/api/
│   ├── routes/
│   │   └── data_cleaning.py       # 新建：数据清洗相关路由
│   └── schemas/
│       └── data_cleaning.py       # 新建：请求/响应模型
```

### 3.2 模型设计要点

- 继承 `backend.infrastructure.database.base.Base`
- 本模块表使用 `pipeline_` 前缀（与项目其他表 `gd2502_` 区分），需在模型中显式设置 `__tablename__`
- 使用 `generate_ulid`
- `dataSetsPath.id` 支持主动设置，不使用 `default=generate_ulid`（根节点可预置固定 id）
- JSON 字段统一使用 `JSONB`

### 3.3 仓储层设计

- 继承 `BaseRepository`，复用 `get_by_id`、`get_all`、`create`、`update`、`delete`
- 扩展方法示例：
  - `DataSetsPathRepository`：`get_children_by_path(id_path)`、`get_tree()`
  - `DataSetsRepository`：`get_by_path_id(path_id)`、`get_list_with_total()`
  - `DataSetsItemsRepository`：`get_by_dataset_id()`、`get_list_with_total()`、`get_by_unique_key()`
  - `ImportConfigRepository`：`get_all()`（配置数量少，无需复杂查询）

### 3.4 API 设计

**路由前缀**：`/api/v1/data-cleaning`

| 资源 | 方法 | 路径 | 说明 |
|------|------|------|------|
| dataSetsPath | GET | /paths | 获取路径树或列表 |
| dataSetsPath | GET | /paths/{id} | 获取单条 |
| dataSetsPath | POST | /paths | 创建 |
| dataSetsPath | PUT | /paths/{id} | 更新 |
| dataSetsPath | DELETE | /paths/{id} | 删除 |
| dataSets | GET | /datasets | 列表（支持 path_id 筛选、分页） |
| dataSets | GET | /datasets/{id} | 获取单条 |
| dataSets | POST | /datasets | 创建 |
| dataSets | PUT | /datasets/{id} | 更新 |
| dataSets | DELETE | /datasets/{id} | 删除 |
| dataSetsItems | GET | /datasets/{dataset_id}/items | 数据项列表（分页、status 筛选） |
| dataSetsItems | GET | /datasets/{dataset_id}/items/{id} | 获取单条 |
| dataSetsItems | POST | /datasets/{dataset_id}/items | 创建 |
| dataSetsItems | PUT | /datasets/{dataset_id}/items/{id} | 更新 |
| dataSetsItems | DELETE | /datasets/{dataset_id}/items/{id} | 删除 |
| importConfig | GET | /import-configs | 列表 |
| importConfig | GET | /import-configs/{id} | 获取单条 |
| importConfig | POST | /import-configs | 创建 |
| importConfig | PUT | /import-configs/{id} | 更新 |
| importConfig | DELETE | /import-configs/{id} | 删除 |

### 3.5 Pydantic Schema 设计

- 请求：`DataSetsPathCreate`、`DataSetsPathUpdate`、`DataSetsCreate`、`DataSetsUpdate`、`DataSetsItemsCreate`、`DataSetsItemsUpdate`、`ImportConfigCreate`、`ImportConfigUpdate`
- 响应：`DataSetsPathResponse`、`DataSetsResponse`、`DataSetsItemsResponse`、`ImportConfigResponse`
- 列表响应：`DataSetsPathListResponse`、`DataSetsListResponse`、`DataSetsItemsListResponse`、`ImportConfigListResponse`（含 `total`、`items`）

## 4. 前端设计

### 4.1 入口与布局

- **独立入口**：新增页面入口「数据清洗」，与现有「动态流程系统」同级或作为子入口
- **布局风格**：与 `frontend/index.html` 一致
  - 左侧：菜单栏（宽度 240px，背景 #304156）
  - 中间：Tab 页区域（点击菜单在中间新增 Tab，可多 Tab 切换、关闭）

### 4.2 菜单结构

| 菜单项 | 说明 |
|--------|------|
| Step00导入管理 | 选择导入数据源、新建导入任务（导入逻辑暂不实现，仅预留界面与配置管理） |
| Step01原始数据管理 | 管理 dataSet 数据（浏览、编辑 dataSets、dataSetsItems） |

### 4.3 实现方式

**采用方案 B：独立页面**  
- 新建 `frontend/data-cleaning.html`，复制 `index.html` 的布局风格
- 左侧菜单包含「Step00导入管理」「Step01原始数据管理」
- 路由：`/static/data-cleaning.html`

### 4.4 子页面设计

#### 4.4.1 Step00导入管理

- 顶部：导入配置列表（ImportConfig CRUD）
- 中部：选择数据源类型、选择目标 dataSet、新建导入任务（按钮可先占位，逻辑后续实现）
- 可选：展示导入历史（后续扩展）

#### 4.4.2 Step01原始数据管理

- 左侧：dataSetsPath 树形结构，点击节点筛选 dataSets
- 右侧上：dataSets 列表（表格，支持新建、编辑、删除）
- 右侧下：选中 dataSet 后，展示 dataSetsItems 列表（表格，支持新建、编辑、删除、status 切换）

## 5. 数据库迁移

### 5.1 Alembic 迁移脚本

新建迁移：`alembic revision -m "add_data_cleaning_tables"`

迁移内容：
1. 创建 `pipeline_data_sets_path`
2. 创建 `pipeline_data_sets`（外键 `path_id` -> `pipeline_data_sets_path.id`）
3. 创建 `pipeline_data_sets_items`（外键 `dataset_id` -> `pipeline_data_sets.id`）
4. 创建 `pipeline_import_config`

### 5.2 外键与级联

- `dataSets.path_id` -> `dataSetsPath.id`：ON DELETE SET NULL 或 RESTRICT（根据业务：删除文件夹时是否允许保留数据集）
- `dataSetsItems.dataset_id` -> `dataSets.id`：ON DELETE CASCADE（删除数据集时级联删除数据项）

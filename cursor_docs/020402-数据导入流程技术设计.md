## 1. 概述

### 1.1 目标

将 `import_feishu_excel.py` 脚本的导入逻辑拆解并重构到主流程代码中，实现：

1. **前端入口**：Step00 导入管理界面增加「执行导入」按钮，读取导入配置并调用后端导入 API
2. **后端导入 API**：接受导入配置 ID，查询配置并执行导入流程
3. **流程代码**：在 `backend/pipeline` 下实现「读取器 → 清洗器 → 入库执行器」三阶段流程
4. **禁止引用 scripts**：所有逻辑在 backend 内重写，不引用 scripts 模块

### 1.2 流程架构

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  原始数据读取器   │ ──► │   数据清洗器     │ ──► │  入库执行器     │
│  (Excel/DB等)   │     │  (按配置选择)    │     │  (通用一套)     │
└─────────────────┘     └─────────────────┘     └─────────────────┘
         │                        │                        │
         ▼                        ▼                        ▼
   配置: filePath          配置: cleaners           配置: dataSetsId
   sheetNames 等           default + sheet 级       校验存在后写入
```

## 3. 后端导入 API

### 3.1 接口定义

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/data-cleaning/import-configs/{config_id}/execute` | 执行导入 |

**路径参数**：`config_id` — 导入配置 ID

**响应**（成功 200）：

```json
    "fail": 0,
    "skipped": 2
```

```json
  "detail": "DataSets 不存在: xxx"
```

## 4. 配置结构定义（import_config.import_config 字段）

### 4.1 完整 JSON 结构

`import_config` 表的 `import_config` 字段（JSONB）结构如下：

```json
{
  "sourceType": "excel",
  "sourcePath": {
    "filePath": "static/rag_source/uat_data/4.1 lsk_副本.xlsx"
  },
  "sheetNames": null,
  "cleaners": {
    "default": "lsk",
    "常见问题及单轮": "sh1128_multi",
    "患者无数据+历史会话+历史Action": "sh1128_history_qa"
  },
  "dataSetsId": "01ARZ3NDEKTSV4RRFFQ69G5FAV"
}
```

### 4.2 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| sourceType | string | 是 | 源类型。当前仅支持 `excel`；`pg` 等后续扩展 |
| sourcePath | object | 是 | 源路径。excel 时为 `{"filePath": "相对项目根路径"}`；数据库时为 `{"tableName": "表A"}` |
| sheetNames | string \| string[] \| null | 否 | Excel 时：`null` 或省略表示解析全部 sheet；`"Sheet1"` 只解析该 sheet；`["Sheet1","Sheet2"]` 只解析列出的多个 sheet |
| cleaners | object | 是 | 清洗器配置，JSON 对象。key 为 sheet 名称，value 为该 sheet 使用的清洗器 key；特殊 key `"default"` 表示未显式配置的 sheet 使用的默认清洗器。示例：`{"default":"lsk","常见问题及单轮":"sh1128_multi"}` |
| dataSetsId | string | 是 | 目标 DataSets 的 ID。若不存在则导入中断 |

### 4.3 路径约定

- `filePath` 以**项目根目录**为基准，如：`static/rag_source/uat_data/xxx.xlsx`
- 后端解析时：`Path(project_root) / filePath`

### 4.4 待办：dataSets 自动创建

**当前版本**：不实现 dataSets 自动创建。配置中的 `dataSetsId` 必须已存在，否则导入中断并返回错误。

**待办**（后续版本）：根据配置自动创建 dataSets，需设计：
- 创建时的 `name`、`path_id`、`input_schema`、`output_schema` 来源
- 与「按 excel/sheet 动态创建」的映射关系
- 与现有 dataSetsPath 树结构的集成

### 5.1 目录结构

```
│   │   └── excel_reader.py     # Excel 读取器（当前唯一实现）

│   │   ├── canonical.py       # CanonicalItem 与转换
│   │   └── impl/

│       └── dataset_item_writer.py   # 入库执行器（写 pipeline_data_sets_items）
```

### 5.2 三阶段流程

#### 5.2.1 原始数据读取器（readers）

**职责**：根据配置读取原始数据，输出为 `(sheet_name, DataFrame)` 的迭代。

**Excel 读取器**（当前唯一实现）：

- 输入：`sourcePath.filePath`、`sheetNames`
- 逻辑：
  - 解析 `filePath`，以项目根目录拼接得到绝对路径
  - 若 `sheetNames` 为 `null` 或省略：迭代所有 sheet
  - 若 `sheetNames` 为字符串：只读取该 sheet，返回单次迭代
  - 若 `sheetNames` 为数组 `["Sheet1","Sheet2"]`：只读取列出的 sheet，按顺序迭代
- 输出：`Iterator[Tuple[str, pd.DataFrame]]`，即 `(sheet_name, df)`

**扩展点**：后续可增加 `pg` 等读取器，根据 `sourceType` 分发。

#### 5.2.2 数据清洗器（cleaners）

**职责**：将 DataFrame 行清洗为统一格式 `CanonicalItem`，再转换为入库结构。

**清洗器选择逻辑**（来自配置 `cleaners`，JSON 对象）：

- 结构：`{ "default": "lsk", "Sheet1": "typeA", "Sheet2": "typeB", ... }`，key 为 sheet 名称，value 为清洗器 key
- 规则：优先用 `cleaners[sheet_name]`，若无则用 `cleaners["default"]`
- 若 `default` 缺失：可约定使用 `"lsk"` 或报错（建议报错，强制配置完整）

**CanonicalItem 结构**（与 scripts 中一致，便于迁移）：

```python
@dataclass
class CanonicalItem:
    current_msg: str = ""
    history_messages: List[Dict[str, str]] = field(default_factory=list)
    response_message: str = ""
    message_id: Optional[str] = None
    patient_id: Optional[str] = None
    doctor_id: Optional[str] = None
    context: Dict[str, Any] = field(default_factory=dict)
    ext: Optional[str] = None
```

**转换**：`canonical_to_dataset_item(CanonicalItem) -> {input, output, metadata}`，对应 dataSetsItems 的 `input`、`output`、`metadata` 字段。

**清洗器实现**：从 scripts 的 `parsers/cleaners` 重写到 `backend/pipeline/cleaners/impl/`，不引用 scripts。首版至少实现 `lsk`，其余按需迁移。

#### 5.2.3 入库执行器（writers）

**职责**：将清洗后的数据写入 `pipeline_data_sets_items` 表。

**输入**：

- `dataset_id`：来自配置的 `dataSetsId`
- `items`：`List[{input, output, metadata}]`
- 可选：`unique_key`、`source`、`status`

**逻辑**：

- 使用 `DataSetsItemsRepository.create()` 逐条或批量插入
- `source` 建议设置为：`excel:{filePath}:{sheet_name}` 或类似格式，便于追溯
- `status` 默认 1（激活）

**前置校验**：调用前需校验 `dataSetsId` 是否存在，不存在则直接返回错误，不执行导入。

## 6. 导入服务主流程（import_service.py）

### 6.1 伪代码

```python
async def execute_import(config_id: str, session: AsyncSession) -> ImportResult:
    # 1. 查询导入配置
    config = await import_config_repo.get_by_id(config_id)
    if not config:
        raise ImportConfigNotFoundError(config_id)
    
    meta = config.import_config or {}
    
    # 2. 校验 dataSetsId
    dataset = await data_sets_repo.get_by_id(meta.get("dataSetsId"))
    if not dataset:
        raise DataSetsNotFoundError(meta.get("dataSetsId"))
    
    # 3. 根据 sourceType 选择读取器
    if meta.get("sourceType") != "excel":
        raise UnsupportedSourceTypeError(meta.get("sourceType"))
    
    reader = ExcelReader(meta)
    writer = DatasetItemWriter(session)
    stats = {"success": 0, "fail": 0, "skipped": 0}
    
    # 4. 迭代 sheet → 清洗 → 入库
    for sheet_name, df in reader.iter_sheets():
        cleaner_key = meta.get("cleaners", {}).get(sheet_name) or meta.get("cleaners", {}).get("default")
        if not cleaner_key:
            raise CleanerNotConfiguredError(sheet_name)
        
        cleaner = get_cleaner_by_type(cleaner_key)
        
        for idx, row in df.iterrows():
            if cleaner.is_empty_row(row, df):
                stats["skipped"] += 1
                continue
            try:
                canonical_items = cleaner.clean(row, df)
                for item in canonical_items:
                    dto = canonical_to_dataset_item(item)
                    await writer.write_item(dataset.id, dto, source=...)
                    stats["success"] += 1
            except Exception as e:
                stats["fail"] += 1
                logger.error("行 %d 处理失败: %s", idx + 2, e)
    
    return ImportResult(**stats)
```

### 6.2 可选：导入前清空

原脚本支持 `clear_before_import`。若需支持，可在配置中增加：

```json
{
  "clearBeforeImport": true
}
```

逻辑：导入前根据 `dataSetsId` 删除该 dataset 下所有 items（或标记废弃）。**当前版本可先不实现**，在「与脚本对比的差异」中说明。

## 7. 与 scripts 脚本的差异与遗漏说明

以下为相对 `import_feishu_excel.py` 的差异或可能遗漏点，供决策是否追加设计：

| 项目 | 脚本行为 | 当前设计 | 建议 |
|------|----------|----------|------|
| 数据存储 | 写入 Langfuse DataSet | 写入 pipeline_data_sets_items | 已按需求设计，无需变更 |
| 多 Excel | 支持一次处理多个 Excel 文件 | 单配置单数据源，一次导入一个 Excel | 多文件需多次执行或多次配置，可接受 |
| 导入前清空 | `clear_before_import` 可清空已有 Items | 未设计 | 可选：配置增加 `clearBeforeImport` |
| DataSet 创建 | 按 excel/sheet 动态创建 Langfuse DataSet | 不创建，仅校验 dataSetsId 存在 | 已标记待办 |
| Schema 设置 | 创建 DataSet 时设置 input/output schema | dataSets 表已有 schema，导入不修改 | 合理 |
| pipeline_log | 无 | QA 文档提到导入日志表 | 建议后续增加 pipeline_log 记录导入结果 |
| 清洗器覆盖 | excel 级 + sheet 级 | 仅 default + sheet 级 | 可满足大部分场景，excel 级可后续扩展 |
| sheet 过滤 | sheet_include_mapping 可只处理部分 sheet | sheetNames 支持 null/单字符串/数组，可多选 sheet | 已支持多选 sheet |

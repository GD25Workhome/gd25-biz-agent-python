## 0. 前置漏洞修复：任务类型与执行器设计（方案二）

### 0.1 问题回顾

- 当前 `batch_task` 仅记录来源表信息、运行参数、执行结果等，**没有记录“任务类型”以及对应的执行入口（后端接口/执行器）**。
- 批次创建完成后，在“运行批次任务”阶段，如果只拿到 `BatchTask` 本身，很难自动推断应该由哪个执行器来处理该任务。

### 0.2 设计目标

- 在不破坏现有“创建模版”结构的前提下，引入**任务类型 + 执行器注册表**机制：
  - **任务类型**：`job_type` 存在批次表 `batch_job` 上，表示“这个批次属于哪一类任务”（如 `embedding`、`data_clean` 等）。
  - **执行器注册表**：运行阶段根据 `job_type` 找到对应的 `BatchTaskExecutor` 实现类，执行单条或整批任务。
- 创建阶段的职责：**在创建批次时写好 `job_type` 字段**，为后续运行阶段提供充足信息。

### 0.3 表结构与模型修改（batch_job 增加 job_type）

1. **表设计层面（补充到 `022603-数据embedding批次表字段设计.md`）**

在批次表 `batch_job` 中新增字段：


| 字段名      | 类型          | 说明                  |
| -------- | ----------- | ------------------- |
| job_type | VARCHAR(64) | 批次任务类型（如 embedding） |


1. **models 层修改：`backend/infrastructure/database/models/batch/batch_job.py`**

在 `BatchJobBusinessMixin` 中新增一列（伪代码示意，实际以 Alembic 迁移为准）：

```python
class BatchJobBusinessMixin:
    """batch_job 业务字段：code、total_count、query_params、job_type。"""

    job_type = Column(
        String(64),
        nullable=False,
        comment="批次任务类型（如 embedding、data_clean 等）",
    )
    code = Column(
        String(64),
        nullable=False,
        unique=True,
        index=True,
        comment="批次编码，唯一",
    )
    total_count = Column(
        Integer,
        nullable=False,
        comment="本批次待 embedding 条数",
    )
    query_params = Column(
        JSON,
        nullable=True,
        comment="查询参数（如筛选条件、分页等）",
    )
```

> 说明：任务类型放在 `batch_job` 上即可，同一批次的子任务类型一致，无需在 `batch_task` 上重复存储，后续执行端先查 `batch_job.job_type` 再处理其下所有 `batch_task`。

1. **仓储层修改：`BatchJobRepository.create` 增加 job_type 参数**

```python
class BatchJobRepository(AuditBaseRepository[BatchJobRecord]):
```

## 1. 需求要点（来自 step3）

### 创建批次任务（模版流程）

1. **调用数据查询接口，返回 task 预创建对象** → 子类实现
2. **生成批次 code** → 模版类默认实现，可被子类覆盖
3. **组装 batch_job、batch_task 的数据库插入对象并落库** → 模版类实现（含 **job_type**，由子类通过 `get_job_type()` 提供，供运行阶段按类型路由执行器）

## 3. 数据结构：task 预创建对象

子类「查询接口」返回的预创建对象，用于模版类组装 `batch_task` 插入数据。

```python
# 建议放在 backend/domain/batch/dto.py 或与模版同包

from dataclasses import dataclass
from typing import Any, Dict, Optional


@dataclass
class TaskPreCreateItem:
    """单条子任务的预创建数据（由子类 query 接口返回）。"""

    source_table_id: Optional[str] = None
    source_table_name: Optional[str] = None
    runtime_params: Optional[Dict[str, Any]] = None
    redundant_key: Optional[str] = None
```

模版类将 `List[TaskPreCreateItem]` 转为多条 `BatchTaskRecord` 并调用 `BatchTaskRepository.create`。

## 4. 关键类与关键代码

### 4.1 创建批次任务模版：`BatchCreateTemplate`

- **模版方法**：`create_batch(session)`，固定四步：查询 → 生成 code → 取 job_type → 组装并落库。
- **抽象方法**：`query_tasks_for_pre_create(session, query_params)`、`get_job_type()`，子类必须实现。
- **钩子方法**：`generate_batch_code()`，模版默认实现（如 ULID/时间戳），子类可覆盖。
- **job_type**：由 `get_job_type()` 返回并写入 `batch_job.job_type`，供运行阶段按类型从执行器注册表路由到对应 `BatchTaskExecutor`。

```python
# 建议路径：backend/domain/batch/create_template.py

from abc import ABC, abstractmethod
from typing import Any, Dict, List, Optional

from sqlalchemy.ext.asyncio import AsyncSession

from backend.infrastructure.database.base import generate_ulid
from backend.infrastructure.database.repository.batch.batch_job_repository import BatchJobRepository
from backend.infrastructure.database.repository.batch.batch_task_repository import BatchTaskRepository
from backend.infrastructure.database.models.batch.batch_job import BatchJobRecord
from backend.infrastructure.database.models.batch.batch_task import BatchTaskRecord

# 预创建 DTO 见上节
from backend.domain.batch.dto import TaskPreCreateItem


class BatchCreateTemplate(ABC):
    """创建批次任务模版：定义「查询 → 生成 code → 组装落库」流程，子类实现查询接口。"""

    def __init__(
        self,
        job_repo: BatchJobRepository,
        task_repo: BatchTaskRepository,
    ) -> None:
        self._job_repo = job_repo
        self._task_repo = task_repo

    async def create_batch(
        self,
        session: AsyncSession,
        query_params: Optional[Dict[str, Any]] = None,
    ) -> BatchJobRecord:
        """
        模版方法：创建批次任务。不可重写整体顺序。
        步骤：1. 查询预创建数据  2. 生成批次 code  3. 取 job_type  4. 组装并持久化 batch_job + batch_task。
        """
        # 1. 子类实现：查询数据，返回预创建 task 列表
        task_items: List[TaskPreCreateItem] = await self.query_tasks_for_pre_create(
            session, query_params
        )
        if not task_items:
            raise ValueError("query_tasks_for_pre_create 返回空列表，无法创建批次")

        # 2. 生成批次 code（模版默认实现，子类可覆盖）
        code = self.generate_batch_code()

        # 3. 子类实现：返回本批次任务类型，供落库及运行阶段按类型路由执行器
        job_type = self.get_job_type()

        # 4. 模版实现：组装 batch_job（含 job_type）、batch_task 并落库
        job = await self._build_and_persist_batch(
            session=session,
            job_type=job_type,
            code=code,
            task_items=task_items,
            query_params=query_params,
        )
        return job

    @abstractmethod
    def get_job_type(self) -> str:
        """
        返回本批次的任务类型（如 embedding、data_clean）。
        子类必须实现；运行阶段将根据 batch_job.job_type 从执行器注册表路由到对应 BatchTaskExecutor。
        """
        ...

    @abstractmethod
    async def query_tasks_for_pre_create(
        self,
        session: AsyncSession,
        query_params: Optional[Dict[str, Any]] = None,
    ) -> List[TaskPreCreateItem]:
        """
        数据查询接口，返回本批次要创建的 task 预创建对象。
        子类必须实现（如查 pipeline_data_items_rewritten 待 embedding 数据）。
        """
        ...

    def generate_batch_code(self) -> str:
        """
        生成批次编码。模版默认实现（如 batch-{ulid}），子类可覆盖以定制规则。
        """
        return f"batch-{generate_ulid()}"

    async def _build_and_persist_batch(
        self,
        session: AsyncSession,
        job_type: str,
        code: str,
        task_items: List[TaskPreCreateItem],
        query_params: Optional[Dict[str, Any]] = None,
    ) -> BatchJobRecord:
        """
        模版内部实现：先创建 batch_job（含 job_type），再逐条创建 batch_task。
        """
        job = await self._job_repo.create(
            job_type=job_type,
            code=code,
            total_count=len(task_items),
            query_params=query_params,
        )
        for item in task_items:
            await self._task_repo.create(
                job_id=job.id,
                source_table_id=item.source_table_id,
                source_table_name=item.source_table_name,
                status="pending",
                runtime_params=item.runtime_params,
                redundant_key=item.redundant_key,
            )
        return job
```

**继承关系**：子类必须实现 `query_tasks_for_pre_create` 与 `get_job_type`；如需自定义编码可覆盖 `generate_batch_code`。

## 5. 基础关系小结


| 类型       | 类名                                           | 与模版关系                    | 职责                                                                                                    |
| -------- | -------------------------------------------- | ------------------------ | ----------------------------------------------------------------------------------------------------- |
| 抽象基类     | `BatchCreateTemplate`                        | -                        | 定义创建流程；实现 code 生成与组装落库；声明 `query_tasks_for_pre_create`、`get_job_type` 抽象方法；落库时写入 `batch_job.job_type` |
| 业务实现（创建） | `PipelineEmbeddingBatchHandler`              | 继承 `BatchCreateTemplate` | 实现 `get_job_type()`（返回 `"pipeline_embedding"`）、`query_tasks_for_pre_create`（查待 embedding 数据）          |
| DTO      | `TaskPreCreateItem`                          | 被模版与子类使用                 | 预创建 task 的数据结构                                                                                        |
| 基础设施     | `BatchJobRepository` / `BatchTaskRepository` | 注入到模版类                   | 与现有 models 交互；`BatchJobRepository.create` 需接收 `job_type` 参数                                           |

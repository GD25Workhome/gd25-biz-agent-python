# Feishu Excel 导入 Langfuse Datasets 技术设计文档

## 1. 设计目标

将 `static/rag_source/uat_data/` 目录下的 Excel 数据导入到 Langfuse Datasets，支持不同 Excel、不同 Sheet 的差异化解析，并统一输出为 DataSet Input/Output Schema 格式。

## 2. 功能需求

### 2.1 数据源

- 读取 `static/rag_source/uat_data/` 下的 Excel 文件（如 `4.1 lsk_副本.xlsx`、`sh-1128_副本.xlsx`）
- 支持多 Sheet 读取

### 2.2 执行步骤

1. **新建 DataSet**：按 `feishu/excelName/sheetName` 命名
2. **设置 Schema**：使用 `DataSet-input-schema.json`、`DataSet-output-schema.json` 作为 input/expected_output 的 JSON Schema
3. **解析 Excel**：按 Sheet 类型选择解析策略
4. **写入 DataSet Items**：将解析后的数据写入对应 DataSet

### 2.3 命名与路径规则

- DataSet 名称格式：`feishu/{excelName}/{sheetName}`
- `excelName`：Excel 文件名（不含扩展名）
- `sheetName`：Sheet 名称
- 示例：`feishu/4.1 lsk_副本/Sheet1`

### 2.4 版本控制

- 版本信息统一放在 DataSet 的 `metadata` 中，避免异常

## 3. 架构设计

### 3.1 目录结构

```
scripts/import_to_datasets/feishu_ceshi_case/
├── 导入设计.md                    # 本文档
├── config.py                      # 配置类
├── import_feishu_excel.py         # 主入口
├── tools/
│   └── create_datasets.py         # 委托入口（兼容旧调用）
├── parsers/                       # 解析器模块
│   ├── __init__.py
│   ├── base.py                    # 解析器基类、责任链基类
│   ├── chain.py                   # 责任链实现
│   └── strategies/                # 具体解析策略
│       ├── __init__.py
│       ├── lsk.py                 # 4.1 lsk 格式
│       └── sh1128.py              # sh-1128 格式
├── dataset/                       # DataSet 操作
│   ├── __init__.py
│   ├── creator.py                 # 创建 DataSet、设置 Schema
│   └── item_writer.py             # 写入 DataSet Items
└── utils/                         # 工具函数
    ├── __init__.py
    └── excel_reader.py            # Excel 读取
```

### 3.2 设计模式

#### 3.2.1 策略模式 + 责任链

- **策略模式**：不同 Sheet 类型对应不同解析策略（如 `LskParser`、`Sh1128Parser`）
- **责任链模式**：字段级处理链，依次完成：
  1. 空值/“无” 处理
  2. 类型转换（int、string、text）
  3. 特殊字段提取（如 message_id）
  4. 组装 DataSet Input/Output 结构

#### 3.2.2 Sheet 解析策略选择

优先级：`sheet_parser_mapping` > `excel_parser_mapping` > `default_parser_type`

- **默认策略**：Config 中配置 `default_parser_type`
- **按 Excel 指定**：`excel_parser_mapping: { "excelName": "parser_type" }`，如 `{"4.1 lsk_副本": "lsk", "sh-1128_副本": "sh1128"}`
- **按 Sheet 覆盖**：`sheet_parser_mapping: { "excelName/sheetName": "parser_type" }`，用于同一 Excel 内不同 Sheet 使用不同策略

### 3.3 数据流

```
Excel 文件 → ExcelReader → 按 Sheet 迭代
    → 策略选择器(Config) → 对应 Parser
    → 责任链(字段格式化) → DataSetItem 基础数据
    → ItemWriter → Langfuse create_dataset_item
```

## 4. Config 配置设计

### 4.1 配置项

| 参数 | 类型 | 说明 |
|------|------|------|
| `data_dir` | Path | Excel 所在目录，默认 `static/rag_source/uat_data` |
| `excel_files` | List[str] | 要导入的 Excel 文件名列表 |
| `sheet_include_mapping` | Dict[str, List[str]] | 非空时仅处理映射中的 Excel 及其列出的 sheet；未在映射中的 Excel 整体跳过。为空则处理全部 |
| `dataset_name_prefix` | str | DataSet 名称前缀，默认 `feishu` |
| `default_parser_type` | str | 默认解析策略类型 |
| `excel_parser_mapping` | Dict[str, str] | `excelName` → `parser_type` 按 Excel 的默认策略 |
| `sheet_parser_mapping` | Dict[str, str] | `excelName/sheetName` → `parser_type` Sheet 级覆盖 |
| `input_schema_path` | Path | Input JSON Schema 文件路径 |
| `output_schema_path` | Path | Output JSON Schema 文件路径 |
| `script_name` | str | 脚本名称，写入 metadata |
| `clear_before_import` | bool | 是否在导入前清空 DataSet 已有 Items（默认 True，实现重复导入覆盖） |

### 4.2 DataSet Metadata 设计

| 字段 | 类型 | 说明 |
|------|------|------|
| `excel_name` | str | 来源 Excel 文件名 |
| `sheet_name` | str | 来源 Sheet 名称 |
| `parser_type` | str | 使用的解析策略类型 |
| `script_name` | str | 导入脚本名称 |
| `import_version` | str | 导入版本（如日期或版本号） |
| `import_time` | str | 导入时间 ISO 格式 |
| `input_schema_ref` | str | Input Schema 引用（$id），供 UI 配置参考 |
| `output_schema_ref` | str | Output Schema 引用（$id），供 UI 配置参考 |

**Schema 说明**：当前 Langfuse Python SDK 的 `create_dataset` 不支持 `input_schema`/`expected_output_schema` 参数，Schema 校验需在 Langfuse UI 中手动配置。Schema 文件路径在 Config 中指定，metadata 中会写入 schema 引用供参考。

## 5. Input/Output Schema 映射

### 5.1 Input Schema（必填 current_msg）

- `current_msg` ← 新会话 / 会话输入
- `history_messages` ← 由 历史会话、历史会话响应 组装为 BaseMessage 数组
- `context` ← 年龄、疾病、血压、症状、用药等作为 context 对象

### 5.2 Output Schema（必填 response_message）

- `response_message` ← 新会话响应 / 供应商响应()
- `flow_msgs` ← 可选，过程消息
- `other_meta_data` ← message_id、ext 等

## 6. 解析策略示例

### 6.1 LSK 策略（4.1 lsk_副本.xlsx）

- 列映射：新会话、新会话响应、ids（message_id 来源）
- 空行判断：新会话、新会话响应 任一为空则跳过

### 6.2 SH1128 策略（sh-1128_副本.xlsx）

- 列映射：会话输入→新会话，供应商响应()→新会话响应，message_id 列
- 空行判断：会话输入、供应商响应() 任一为空则跳过

## 7. 实现要点

1. **Config 单例**：所有参数从 Config 读取，无需命令行传参
2. **可扩展**：新增解析策略只需实现 `BaseSheetParser` 并注册
3. **幂等性**：同一 DataSet 多次运行可覆盖或追加（由 Langfuse 行为决定）
4. **错误处理**：单行解析失败不影响其他行，记录失败数

## 8. 重复导入

- **Config.clear_before_import**（默认 True）：每次运行前清空目标 DataSet 中已有 Items，再导入新数据，实现覆盖而非追加
- 若设为 False：每次运行会向已有 DataSet 追加新 Items（数据会重复累积）

## 9. 会话输入/供应商响应多轮提取改造分析

> 针对 `sh-1128_副本` 的「常见问题及单轮」Sheet 中，**会话输入**、**供应商响应()** 字段的特殊数据格式，实现「一行 Excel 对应多条 DataSet Item」的兼容与可扩展方案分析。

### 9.1 需求背景

#### 9.1.1 数据格式说明

**会话输入** 字段原始文本示例：

```
Q：小悦您好！我想约龙医生，做造影
A：您有和医生提前预约吗？
Q：没有
A：了解了，方便告知下患者就诊姓名、诊疗卡号、住院时间、具体什么情况吗？
```

- 格式：多轮 Q/A 交替，`Q：` 为用户提问，`A：` 为系统追问/澄清
- 语义：第一次提问为 Q1，回答为 A1；第二次提问为 Q2，历史为 Q1+A1，回答为 A2
- **一条 Excel 行对应多轮对话，需拆分为多条 Item**

**供应商响应** 字段原始文本示例：

```
A:你可以先查看龙医生近一个月的门诊排班信息，了解他的出诊时间安排。
[未知类型(tool_doctor_schedule): {"type": "tool_doctor_schedule", "tool_doctor_schedule": {"start_day": "2025-11-27", "end_day": "2025-12-24"}}]
需要说明的是，我无法直接帮你预约挂号...
A:我是医生的数字分身，可以智能响应您的健康需求...
```

- 格式：多个 `A:` 块，块内可能包含 `[未知类型(tool_xxx): {...}]` 等 tool 调用标记
- 语义：每个 `A:` 块对应一轮供应商响应，与「会话输入」中的 Q 一一对应
- **需按 A 块拆分，与 Q 配对**

**message_id** 字段：

- Excel 中可能有多行 message_id（如换行分隔），需与拆分后的 Item 一一对应

#### 9.1.2 目标数据模型

| Item | current_msg | history_messages | response_message | message_id |
|------|-------------|-----------------|------------------|------------|
| 1 | Q1 | [] | A1 | message_id_1 |
| 2 | Q2 | [Q1, A1] | A2 | message_id_2 |

### 9.2 当前架构兼容性分析

#### 9.2.1 数据流现状（1 行 → 1 Item）

```
Excel 行 → extract_row_data() → row_data (单条)
         → chain.process(row_data) → DataSetItemData (单条)
         → writer.write_item(item)
```

- **extract_row_data**：按列提取，返回单个 `Dict`，`current_session`、`response` 各一个
- **to_dataset_item**：`row_data` → 单个 `DataSetItemData`
- **主循环**：`for idx, row in df.iterrows()`，每行调用一次 `writer.write_item`

#### 9.2.2 不兼容点

| 环节 | 现状 | 新需求 | 兼容性 |
|------|------|--------|--------|
| extract_row_data | 返回单条 row_data | 需解析出多组 (Q, history, A) | ❌ 结构不兼容 |
| to_dataset_item | 1 row_data → 1 Item | 1 row_data → N Items | ❌ 输出数量不兼容 |
| 主循环 | 每行 1 次 write_item | 每行 N 次 write_item | ⚠️ 需扩展 |
| message_id | 单值 extract_message_id | 多值拆分并一一对应 | ❌ 需新逻辑 |

### 9.3 可扩展方案设计

#### 9.3.1 方案概览

采用 **「解析策略扩展 + 主循环适配」** 思路，在不破坏现有 LSK、SH1128 单行单 Item 逻辑的前提下，为 SH1128 的「常见问题及单轮」Sheet 增加多 Item 解析能力。

#### 9.3.2 扩展点设计

**1. 解析层：新增「多 Item 解析」能力**

在 `BaseSheetParser` 中增加可选方法：

```python
def extract_items_from_row(self, row: pd.Series, df: pd.DataFrame) -> List[Dict[str, Any]]:
    """
    从单行提取多条 Item 的 row_data 列表。
    默认实现：调用 extract_row_data + to_dataset_item，返回单条包装为 List。
    子类重写：实现多轮 Q/A 拆分，返回多条。
    """
    row_data = self.extract_row_data(row, df)
    if self.is_empty_row(row, df):
        return []
    return [row_data]  # 默认 1:1
```

**Sh1128MultiParser**（或扩展 Sh1128Parser）重写逻辑：

- 解析「会话输入」：按 `Q：`、`A：` 拆分，得到 `[(Q1, A1_pre), (Q2, A2_pre), ...]`（A1_pre 为追问，可并入 history）
- 解析「供应商响应」：按 `A:` 或正则 `^A\s*[:：]` 拆分为 `[A1, A2, ...]`
- 解析 message_id：支持多行/多值，得到 `[id1, id2, ...]`
- 组装：`[(Q1, [], A1, id1), (Q2, [Q1,A1], A2, id2), ...]`，每条转为 `row_data` 结构

**2. 责任链：支持 List 输入**

- `FieldProcessorChain.process` 当前签名：`process(row_data) -> Optional[DataSetItemData]`
- 扩展方案 A：新增 `process_multi(row_data_list) -> List[DataSetItemData]`，内部对每条调用现有 `process`
- 扩展方案 B：`process` 支持 `Union[Dict, List[Dict]]`，返回 `Union[DataSetItemData, List[DataSetItemData]]`，主循环统一处理

**3. 主循环适配**

```python
# 伪代码
items_data = parser.extract_items_from_row(row, df)
for item_row_data in items_data:
    item = chain.process(item_row_data)
    if item:
        writer.write_item(dataset_name, item)
        sheet_stats["success"] += 1
```

- 空行：`extract_items_from_row` 返回 `[]`，主循环 `continue`
- 单 Item：返回 `[row_data]`，行为与现有一致

**4. message_id 多值提取**

新增 `extract_message_ids(value) -> List[str]`：

- 支持换行、分号、逗号等分隔
- 支持 `messageId: xxx` 格式的批量解析
- 与 Q/A 拆分结果按索引一一对应，不足时用 `None` 或最后一项填充（可配置）

#### 9.3.3 解析规则细化

**会话输入解析规则**：

| 规则 | 说明 |
|------|------|
| 分隔符 | `Q：`、`Q:`、`A：`、`A:`（中英文冒号） |
| 块顺序 | 严格按 Q→A→Q→A 交替，否则按出现顺序 |
| 追问处理 | 第一次 `A` 为系统追问，作为 history 的一部分；最后一次 `A` 前的 `Q` 为当前轮用户提问 |
| 多轮提取 | 每轮 (当前 Q, history, 对应 A) 生成一条 row_data |

**供应商响应解析规则**：

| 规则 | 说明 |
|------|------|
| 分隔符 | `A:` 或 `A：` 开头的新块 |
| Tool 标记 | `[未知类型(tool_xxx): {...}]` 保留在对应 A 块内，不单独拆分 |
| 块数 | 与「会话输入」的 Q 数量一致，按顺序一一对应 |

**message_id 解析规则**：

| 规则 | 说明 |
|------|------|
| 多行 | `\n`、`\r\n` 分隔 |
| 多值 | `;`、`,` 分隔（可选） |
| 格式 | 支持 `messageId: xxx`、纯 UUID |
| 对应 | 按索引与 Item 对应，超出部分忽略，不足部分用 `None` 或最后一项 |

### 9.4 实现路径建议

| 阶段 | 内容 | 涉及文件 |
|------|------|----------|
| 1 | 新增 `extract_message_ids`、`parse_qa_blocks`、`parse_response_blocks` 工具函数 | `parsers/base.py` 或 `parsers/utils.py` |
| 2 | `BaseSheetParser.extract_items_from_row` 默认实现，保持现有策略 1:1 | `parsers/base.py` |
| 3 | 新增 `Sh1128MultiParser` 或扩展 `Sh1128Parser`，重写 `extract_items_from_row` | `parsers/strategies/sh1128.py` 或新文件 |
| 4 | Config 增加 `sheet_parser_mapping` 或新 parser_type，将「常见问题及单轮」指向多 Item 解析器 | `config.py` |
| 5 | 主循环：`extract_items_from_row` → 遍历 → `chain.process` → `write_item` | `import_feishu_excel.py` |

### 9.5 兼容性保障

- **LSK、现有 SH1128 单行逻辑**：`extract_items_from_row` 默认返回 `[extract_row_data(...)]`，主循环仅多一层遍历，行为不变
- **责任链**：不修改 `process` 签名，仅主循环对多条 `row_data` 分别调用
- **Config**：通过 `sheet_parser_mapping` 或 `excel_parser_mapping` 按 Sheet 选择解析器，不影响其他 Sheet

### 9.6 风险与边界

| 风险 | 说明 | 缓解 |
|------|------|------|
| Q/A 数量与 A 块数量不一致 | 会话输入 3 轮 Q，供应商响应 2 个 A 块 | 按最小数量对齐，或记录告警、跳过异常行 |
| message_id 数量与 Item 数量不一致 | 2 个 message_id，3 条 Item | 不足用 None，超出忽略；可配置填充策略 |
| 格式变异 | 分隔符、空格、换行不统一 | 解析前做 `strip`、统一正则，必要时支持配置化 |

## 2. 入口与请求/响应

### 2.1 接口定义

| 项目 | 取值 |
|------|------|
| 方法 | POST |
| 路径 | `/datasets/{dataset_id}/items/rewritten/execute` |
| 请求体 | `RewrittenExecuteRequest` |
| 响应体 | `RewrittenExecuteResponse` |

### 2.2 请求参数（RewrittenExecuteRequest）

| 字段 | 类型 | 说明 |
|------|------|------|
| `item_ids` | `Optional[List[str]]` | 指定要清洗的数据项 ID 列表；若提供则忽略 `query_params` |
| `query_params` | `Optional[RewrittenExecuteQueryParams]` | 筛选条件，命中多少执行多少；显式传 `{}` 表示无筛选（命中该 dataset 下全部） |

约束：

- 至少提供 `item_ids` 或 `query_params` 之一。
- 若提供 `item_ids` 则不能为空列表。

**RewrittenExecuteQueryParams**（与 `data_sets_items` 查询条件对应）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `status` | `Optional[int]` | 1=激活，0=废弃 |
| `unique_key` | `Optional[str]` | unique_key（包含） |
| `source` | `Optional[str]` | source（包含） |
| `keyword` | `Optional[str]` | 在 input/output/metadata 中搜索 |

### 2.3 响应（RewrittenExecuteResponse）

| 字段 | 类型 | 说明 |
|------|------|------|
| `success` | bool | 是否成功 |
| `message` | str | 提示信息（如「批次已创建，任务将异步执行」） |
| `batch_code` | str | 批次编码（格式 `YYYYMMDDHHmmss`） |
| `total` | int | 本批次实际创建的任务数（init 记录数） |

## 3. 总体流程概览

```
前端/调用方
    │
    ▼ POST /datasets/{dataset_id}/items/rewritten/execute
┌─────────────────────────────────────────────────────────────────┐
│ execute_rewritten_endpoint                                       │
│   参数校验 → 查 dataset 存在 → create_rewritten_batch → commit   │
└─────────────────────────────────────────────────────────────────┘
    │
    ├─ 同步返回：RewrittenExecuteResponse(batch_code, total)
    │
    ▼ 后台（与请求解耦）
┌─────────────────────────────────────────────────────────────────┐
│ rewritten_worker_loop（main.py lifespan 中启动，当前代码中已注释）│
│   轮询 get_init_records → update_status(processing) → 并行执行   │
└─────────────────────────────────────────────────────────────────┘
    │
    ▼ 每条 init 记录
┌─────────────────────────────────────────────────────────────────┐
│ run_one_rewritten(rec)                                           │
│   查 DataSetsItemsRecord → _run_one(item_record, ..., rec.id)    │
└─────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────┐
│ _run_one：build_state_from_record → graph.ainvoke(FlowState)     │
│   Flow: rewritten_agent_node → update_rewritten_data_node → END  │
│   update_rewritten_data_func 按 rewritten_record_id 更新 DB       │
└─────────────────────────────────────────────────────────────────┘
```

## 4. 接口内流程（execute_rewritten_endpoint）

1. **参数校验**
   - 至少提供 `item_ids` 或 `query_params`；若为 `item_ids` 则非空。
2. **数据集校验**
   - `DataSetsRepository.get_by_id(dataset_id)`，不存在则 404。
3. **创建批次与任务**
   - 调用 `create_rewritten_batch(dataset_id, session, item_ids=..., query_params=...)`：
     - 按 `item_ids` 或 `query_params` 从 **pipeline_data_sets_items** 查待处理记录；
     - 生成 `batch_code`（当前时间 `YYYYMMDDHHmmss`）；
     - 在 **pipeline_rewritten_batches** 插入一条批次记录；
     - 在 **pipeline_data_items_rewritten** 为每条来源记录创建 `status=init` 的记录（存在 init/processing/success 则跳过）；
     - 返回 `RewrittenBatchCreateResult(batch_code, total)`。
4. **提交与返回**
   - `session.commit()`，返回 `RewrittenExecuteResponse(success=True, message="批次已创建，任务将异步执行", batch_code=..., total=...)`。

---

## 5. create_rewritten_batch 详细逻辑

- **入参**：`dataset_id`、`session`、`item_ids` 或 `query_params`。
- **数据来源**：
  - `item_ids` 非空：`DataSetsItemsRepository.get_by_ids(dataset_id, item_ids)`；
  - 否则：`DataSetsItemsRepository.get_list_by_conditions(dataset_id, **query_params)`。
- **批次**：
  - `batch_code = datetime.now().strftime("%Y%m%d%H%M%S")`；
  - `create_params` 由 `_build_create_params(dataset_id, item_ids, query_params)` 生成（含 `dataset_id`、`mode`、`item_ids` 或 `query_params`）；
  - `RewrittenBatchRepository.create(batch_code, total_count, create_params)` 写入 **pipeline_rewritten_batches**。
- **改写任务**：
  - `DataItemsRewrittenRepository.create_init_batch(records, batch_code, dataset_id)`：
    - 对每条 `DataSetsItemsRecord`，若同 `source_dataset_id`+`source_item_id` 已有 status 为 init/processing/success 则跳过；
    - 否则 `create(source_dataset_id=dataset_id, source_item_id=record.id, status=init, batch_code=batch_code)` 写入 **pipeline_data_items_rewritten**。
- **返回**：`RewrittenBatchCreateResult(batch_code, total=created)`（created 为实际新增条数）。

---

## 6. 后台 Worker（rewritten_worker_loop）

- **启动位置**：`backend/main.py` 的 `lifespan` 中「6. 启动 Rewritten 后台 Worker」；当前实现中 **`asyncio.create_task(rewritten_worker_loop())` 已被注释**，因此 Worker 未真正运行，需取消注释后才会异步消费 init 记录。
- **逻辑概要**：
  1. 每 `POLL_INTERVAL_SECONDS`（2 秒）轮询；
  2. `DataItemsRewrittenRepository.get_init_records(limit=BATCH_SIZE)`（BATCH_SIZE=10）拉取 `status=init` 的记录；
  3. 将这批记录的 `status` 更新为 `processing` 并 commit；
  4. 使用 `asyncio.Semaphore(MAX_CONCURRENT)`（MAX_CONCURRENT=4）并行执行 `run_one_rewritten(rec)`；
  5. 若某条执行抛错，则 `_mark_failed(record_id, reason)` 将对应记录置为 `failed`。

---

## 7. 单条任务执行（run_one_rewritten → _run_one）

- **run_one_rewritten(rec: DataItemsRewrittenRecord)**  
  - 用 `rec.source_dataset_id`、`rec.source_item_id` 查 **pipeline_data_sets_items**（`get_by_dataset_and_item_id`）；  
  - 若来源记录不存在或 source 为空，则将该条 **pipeline_data_items_rewritten** 更新为 `status=failed` 并写入 `execution_metadata`；  
  - 否则取得 `FlowManager.get_flow(FLOW_KEY)`（`FLOW_KEY = "rewritten_data_service_agent"`），调用 `_run_one(item_record, dataset_id, graph, rewritten_record_id=rec.id)`。

- **_run_one(record, dataset_id, graph, rewritten_record_id)**  
  - 生成 `trace_id`、`session_id`；  
  - `build_state_from_record(record, dataset_id, session_id, trace_id, rewritten_record_id)` 从 **pipeline_data_sets_items** 的 `input`/`output`/`metadata` 构建 `FlowState`（含 `prompt_vars.rewritten_record_id`）；  
  - `graph.ainvoke(initial_state, config)` 执行流程；  
  - 流程内 **update_rewritten_data_node** 调用 `update_rewritten_data_func`，根据 `rewritten_record_id` 更新 **pipeline_data_items_rewritten** 对应记录（改写结果、status=success/failed、execution_metadata 等）。

---

## 8. 流程与节点（rewritten_data_service_agent）

- **配置**：`config/flows/pipeline_step2/flow.yaml`  
  - **entry_node**：`rewritten_agent_node`  
  - **节点**：  
    - `rewritten_agent_node`（agent）：调用 LLM，产出改写结果写入 state（如 `edges_var`）；  
    - `update_rewritten_data_node`（function_key: `update_rewritten_data_func`）：从 state 读取改写结果与 `rewritten_record_id`，更新 **pipeline_data_items_rewritten**。  
  - **边**：`rewritten_agent_node → update_rewritten_data_node → END`。

- **update_rewritten_data_func**（`backend/domain/flows/implementations/update_rewritten_data_func.py`）：  
  - 从 state 取 `rewritten_record_id`（必填，否则抛错）；  
  - 从 `edges_var` 解析有效改写字段，决定 `status=success` 或 `status=failed` 及 `execution_metadata`；  
  - `DataItemsRewrittenRepository.get_by_id(rewritten_record_id)` 后 `repo.update(rewritten_record_id, **kwargs)` 写回 DB。

## 9. 涉及数据表汇总

| 表名 | 模型 | 说明 |
|------|------|------|
| **pipeline_data_sets** | （DataSets 相关模型） | 数据集；接口仅用于校验 dataset 是否存在 |
| **pipeline_data_sets_items** | `DataSetsItemsRecord` | 原始数据项；执行清洗的数据来源，供 build_state_from_record 与 Worker 按 source_item_id 反查 |
| **pipeline_rewritten_batches** | `RewrittenBatchRecord` | 批次表；每次执行创建一条，存 batch_code、total_count、create_params |
| **pipeline_data_items_rewritten** | `DataItemsRewrittenRecord` | 改写结果与执行状态；先由 create_init_batch 写入 init，Worker 改为 processing，flow 结束时由 update_rewritten_data_func 更新为 success/failed 并写入改写字段 |

---

## 10. 表结构与状态流转

### 10.1 pipeline_rewritten_batches

| 字段 | 类型 | 说明 |
|------|------|------|
| id | String(50) | 主键 ULID |
| batch_code | String(100) | 批次编码，唯一，格式 YYYYMMDDHHmmss |
| total_count | Integer | 本批次数据项总量 |
| create_params | JSONB | 创建参数（dataset_id, mode, item_ids/query_params） |
| status | String(20) | 可选，批次级状态 |
| created_at / updated_at | DateTime(tz) | 创建/更新时间 |

### 10.2 pipeline_data_items_rewritten（与执行流程相关的核心字段）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | String(50) | 主键 ULID；作为 rewritten_record_id 传入 flow |
| source_dataset_id | String(100) | 来源 dataSets.id |
| source_item_id | String(100) | 来源 dataItems.id |
| batch_code | String(100) | 所属批次 |
| status | String(20) | **init → processing → success / failed** |
| execution_metadata | JSONB | 失败原因、stage 等 |
| scenario_description, rewritten_question, rewritten_answer, rewritten_rule 等 | Text/JSONB | 由 update_rewritten_data_func 写入 |
| trace_id | String(100) | 流程 traceId |
| created_at / updated_at | DateTime(tz) | 创建/更新时间 |

**状态流转**：

- **init**：create_init_batch 创建。
- **processing**：Worker 拉取后更新。
- **success**：update_rewritten_data_func 在 edges_var 有效时写入。
- **failed**：update_rewritten_data_func 在无有效改写时写入，或 run_one_rewritten/_mark_failed 在异常时写入。

### 10.3 pipeline_data_sets_items（与本接口相关的字段）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | String(50) | 主键，对应 source_item_id |
| dataset_id | String(50) | 所属数据集 |
| input / output / metadata | JSONB | 供 build_state_from_record 构建 prompt_vars |
| status, unique_key, source | 等 | 供 query_params 筛选 |

## 11. 依赖关系简图

```
execute_rewritten_endpoint
  → DataSetsRepository (校验 dataset)
  → create_rewritten_batch
       → DataSetsItemsRepository (get_by_ids / get_list_by_conditions)
       → RewrittenBatchRepository (create)
       → DataItemsRewrittenRepository (create_init_batch)

rewritten_worker_loop
  → DataItemsRewrittenRepository (get_init_records, update_status)
  → run_one_rewritten
       → DataSetsItemsRepository (get_by_dataset_and_item_id)
       → _run_one
            → build_state_from_record(DataSetsItemsRecord)
            → FlowManager.get_flow("rewritten_data_service_agent")
            → graph.ainvoke
                 → update_rewritten_data_func
                      → DataItemsRewrittenRepository (get_by_id, update)
```

## 12. 当前实现注意点

1. **Worker 未启动**：`main.py` 中 `worker_task = asyncio.create_task(rewritten_worker_loop())` 被注释，因此创建批次后不会有后台消费；若要异步执行，需取消该注释并保证只创建一次 task。
2. **单数据集**：当前接口路径为 `/datasets/{dataset_id}/items/rewritten/execute`，创建参数中仅带 `dataset_id`；多数据集支持见 `021102-数据清洗接口多数据集支持改造设计.md`。
3. **幂等与去重**：create_init_batch 会跳过同 source 下已存在 init/processing/success 的项，避免同一来源重复下发改写任务。

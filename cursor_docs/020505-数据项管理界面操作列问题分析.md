# 数据项管理界面操作列问题分析

## 问题描述

在 data-cleaning 页面的数据项管理界面中，存在两个问题：

1. **删除按钮文字被隐藏**：操作列中的「删除」二字不可见，仅显示图标（SVG）
2. **按钮列样式与其它界面不一致**：与原始数据管理、导入管理界面的操作列视觉效果不同

## 对比分析

### 三个界面的操作列配置

| 界面 | 组件 | 操作列宽度 | 按钮数量 | 按钮内容 |
|------|------|-----------|----------|----------|
| 导入管理 | pipeline_import_manage.js | 280px | 4 个 | 执行导入、复制、编辑、删除 |
| 原始数据管理 | pipeline_set_view.js | 200px | 3 个 | 数据项、编辑、删除 |
| **数据项管理** | **pipeline_dataset_items.js** | **160px** | **3 个** | **废弃/激活、编辑、删除** |

### 数据项管理表格列结构

```
expand(48) + ID(180) + unique_key(min-width 120) + source(min-width 120) + 状态(80) + 操作(160)
```

### 原始数据管理表格列结构

```
expand(48) + ID(180) + 名称(min-width 120) + 创建时间(160) + 操作(200)
```

---

## 问题一：删除按钮文字被隐藏

### 根本原因：操作列宽度不足

数据项管理的操作列宽度为 **160px**，是三个界面中最窄的，但需要容纳 3 个 `el-button link` 按钮：

- **废弃/激活**：约 4 个汉字 + 按钮内边距，约 50–60px
- **编辑**：2 字 + 图标(12px) + 内边距，约 50px
- **删除**：2 字 + 图标(12px) + 内边距，约 50px

粗略估算总需求约 **150–170px**，在 160px 下列宽处于临界或不足状态。

### 次要因素

1. **Element Plus 表格 cell 的 overflow 行为**：`.el-table__cell .cell` 默认可能带有 `overflow: hidden` 或类似裁剪，超出部分会被隐藏。
2. **fixed 列与横向滚动**：数据项表格的 `unique_key`、`source` 列内容较长（如 `excel:static/rag_source/uat_data/sh-1128_副本.xlsx:常见问题及单轮`），表格总宽度大，易触发横向滚动。`fixed="right"` 的列会固定在右侧，其宽度由 `width="160"` 决定，若内容超出 160px，多余部分会被裁剪。
3. **white-space: nowrap 的影响**：为防止换行，操作列内容被包在 `<span style="white-space:nowrap;">` 中，内容不会换行，只会横向延伸，更容易超出列宽并被裁剪。

### 结论

操作列 `width="160"` 不足以稳定容纳「废弃/激活 + 编辑 + 删除」三个按钮，导致最右侧的「删除」文字被裁剪，仅剩图标可见。

## 问题二：按钮列样式与其它界面不一致

### 原因分析

#### 1. 操作列宽度差异

- 数据项管理：**160px**
- 原始数据管理：**200px**
- 导入管理：**280px**

数据项管理的操作列明显更窄，视觉上更拥挤。

#### 2. 容器与布局差异

- **数据项管理**：作为独立 Tab 打开，组件占满整个 tab-pane，表格在 `flex:1; overflow:auto; padding:12px` 的容器内。
- **原始数据管理**：左右分栏（左侧 path 树 260px + 右侧数据集列表），表格在右侧 `flex:1` 区域，容器 `padding:12px`。

两者表格所在容器宽度不同，但更直接影响样式的是**操作列宽度**，而非外层容器。

#### 3. 按钮数量与内容差异

- 数据项管理有**状态切换**（废弃/激活），按钮文字随状态变化，且为 4 个字，占用空间更大。
- 原始数据管理为「数据项、编辑、删除」，其中「数据项」为 3 个字。
- 导入管理有 4 个按钮，但操作列宽度为 280px，空间充足。

#### 4. 表格列数差异

数据项管理表格有 **6 列**（含 expand），原始数据管理有 **5 列**。数据项管理的 `unique_key`、`source` 使用 `min-width`，在宽屏下会拉长，进一步压缩固定列（如操作列）的视觉空间。

### 结论

样式不一致主要来自**操作列宽度配置不统一**：数据项管理使用 160px，而其它界面使用 200px 或 280px，导致按钮拥挤、文字被裁切，观感与其它界面不同。

---

## 修复建议（供后续实现参考）

1. **调整操作列宽度**：将数据项管理的操作列由 160px 调整为 200px 或以上，与原始数据管理对齐。
2. **统一操作列宽度规范**：在 data-cleaning 相关组件中约定操作列最小宽度（如 200px），避免类似问题。
3. **验证 fixed 列在横向滚动下的表现**：在 `unique_key`、`source` 内容较长时，确认操作列内容完整可见。
